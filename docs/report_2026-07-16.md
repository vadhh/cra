### 2b. Registry-Driven Classifier Configuration (Priority 2 - `9164835`)
- **Classifier Registry Wiring:** Refactored classifier settings from hardcoded dicts in `detector_distilbert.py` to a registry-driven structure. Added `classifier` blocks (hypothesis, positive/negative keywords, competing profiles, threshold, status) to all 56 profiles in `registry_v1.json` (11 validated, 45 draft).
- **Bug Fix:** Resolved a dormant bug where `load_doc_type_specs()` and `load_keyword_doc_types()` read from a legacy unused `ProfileManager`. Rewired both functions to fetch from `registry_v1.json` via `profile_registry.py`, ensuring all 56 profiles are dynamically loaded and processed in NLI and keyword fallbacks.
- **Coverage:** Added integration checks in `tests/test_registry_classifier.py`.

### 2c. Shadow Code Hygiene (`ac1bbc1`)
- **Dead Code Removal:** Triaged and removed a duplicate module-level `_keyword_doc_type` function which was shadowing the active implementation. This shadow function referenced an undefined global (`_KEYWORD_DOC_TYPES`), posing a latent `NameError` risk if ever reached.

### 2d. Multi-Factor Classifier Confidence Gate (Priority 3 - `9561b3a`)
- **Multi-Factor Evaluator:** Replaced the single-variable confidence check (absolute NLI score < 0.70) with a multi-factor logic in `worker._needs_confirmation()`. It now triggers on:
  - NLI confidence < `LDV_CLASSIFIER_CONFIRMATION_THRESHOLD` (default `0.70`) -> `low_confidence`.
  - NLI confidence < `LDV_CLASSIFIER_HIGH_CONFIDENCE_THRESHOLD` (default `0.85`) and top-two candidate margin < `LDV_CLASSIFIER_MINIMUM_MARGIN` (default `0.15`) -> `ambiguous_margin`.
  - Disagreements between keyword detection and NLI or generic-to-specific overrides -> `keyword_nli_disagreement`.
- **Compatibility:** Kept writing `error_message="low_confidence"` to the database to avoid breaking the frontend UI, but logged the exact trigger reason for developer triage.
- **Configuration:** Documented the new environment variables in `CLAUDE.md`.

### 2e. Portal/PDF Report Parity (Priority 4 - `e3feb70`)
- **PDF Resolution Bug Fix:** Switched profile resolution in `pdf_report.py` from legacy `ProfileManager` to `profile_registry`, ensuring the version is correctly resolved instead of defaulting to "N/A" for the 45 new profiles.
- **NameError Fix:** Resolved a latent runtime crash by importing `logger` into `pdf_report.py` (which was referenced in an error handling branch but never imported).
- **Parity Table:** Added a Score Breakdown table to render `layer3.breakdown` (matching the portal UI layout).
- **Tuning Status and Disclaimer:** Stopped printing a hardcoded `"validation_status": "Validated"` in the PDF. It now outputs the dynamic registry status (`validated`/`draft`) and appends a warning note: `"legal clause-list approval not yet established"`.
- **Coverage:** Added `tests/test_report_parity.py` which uses `PyMuPDF` to verify the 7 required parity dimensions.

### 2f. Hardened Translation Integration & Log Leak Defenses (Priority 5 - `9d58b7d`)
- **Offline Claim Correction:** Updated `CLAUDE.md` and logging statements to reflect that `LDV_REMOTE_TRANSLATION=local` does not run Marian MT completely locally anymore; it calls the external LightML microservice via HTTP POST.
- **Translator Hardening:**
  - Enforced HTTPS by default for `LIGHTML_TRANSLATOR_URL` (insecure HTTP is restricted and requires setting `LDV_LIGHTML_ALLOW_INSECURE=1`).
  - Added optional Bearer API key authentication (`LDV_LIGHTML_API_KEY`).
  - Implemented payload character size limit (`LDV_LIGHTML_MAX_CHARS`, default `50000`).
  - Sanitized exception logging to output type and message only, preventing potential leak of sensitive contract text from response echo.
  - Added health status probe in `GET /health` for the translator.
- **Coverage:** Added 9 new tests covering authorization, payload restrictions, TLS checks, and safe error handling.
