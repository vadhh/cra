## 3. Phase 1 — Translator Integration Guard

**Objective:** Ensure external translation is disabled by default; English documents bypass translation; external calls only when explicitly enabled.

**Change:** [`ldv-backend/translator_client.py`](file:///home/stardhoom/LDV/ldv-backend/translator_client.py)

```diff
+ # External translation is DISABLED BY DEFAULT.
+ if os.getenv("EXTERNAL_TRANSLATION_DISABLED", "1") == "1":
+     return text   # fast path, no network call
+ url = os.getenv("LIGHTML_TRANSLATOR_URL", "")
+ if not url:
+     return text   # URL not configured
```

**Behaviour now:**
- `EXTERNAL_TRANSLATION_DISABLED` defaults to `"1"` → external call never fires on a fresh deployment
- To enable: set `EXTERNAL_TRANSLATION_DISABLED=0` **and** `LIGHTML_TRANSLATOR_URL=<url>`
- Offline mode is unaffected (in-process translator is a separate path)

> ⚠️ **Ilham action:** Confirm that the lightml-translator microservice URL is correctly set in the production `.env` before enabling.
