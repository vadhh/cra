## 1. Summary of Changes

### 1a. NLI Classifier Low-Confidence Gate (Issue #9)
- **Worker interception:** Background worker (`worker.py`) now catches document type classifications where the NLI zero-shot confidence falls below `0.70` (and no manual override is provided).
- **Retryable status:** Low-confidence jobs are saved as `status="retryable"` with `error_message="low_confidence"`.
- **API metadata extraction:** `api_result` endpoint restricts report details while returning the classification candidates and confidence scores.
- **Frontend alert & selection prompt:** `index.html` intercepts this status and shows a clean alert prompt presenting the top detected candidates, a document profile dropdown pre-selected with the highest candidate, and a **Confirm & Analyze** button.
- **Retry override support:** The `/retry` endpoint now accepts `{ "type": "..." }` and passes it to the worker to force a re-run with the confirmed profile.
- **Coverage:** Verified via a new integration test: `ldv-backend/tests/test_low_confidence_gate.py`.
